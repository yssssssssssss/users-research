## 2. 任务执行文本流

```text
[创建任务]
    ↓
[初始化 ResearchTaskState]
    ↓
[runTask]
    ↓
[Problem Decomposer]
    ↓
[Experience Model Analysis]
    ↓
[External Search]（可选）
    ↓
[Vision MoE]（可选）
    ↓
[Persona Sandbox]（可选）
    ↓
[Judgment Synthesizer]
    ↓
[Candidate Outputs]
    ↓
[Gate / blockedReasons]
    ↓
[Generate Report]
    ↓
[Review / Rerun / Persist]
```

```text
├── Vision 评审

└── 多模型复核

模拟能力
└── Persona Sandbox

降级能力
├── Problem fallback
├── Vision weak fallback
├── Judgment fallback
└── External search fallback
```

## 4. 适合继续转绘的建议

如果后续要转成正式图，建议拆成 3 张：

1. **老板版**
   - 只保留：任务中心 → AI 引擎 → 结果 / 报告 / 审核 / 数据留存

2. **研发版**
   - 强调：web / api / taskService / orchestrator / gateway / db

3. **数据流版**
   - 强调：ResearchTaskState 如何从创建到报告一路演化
