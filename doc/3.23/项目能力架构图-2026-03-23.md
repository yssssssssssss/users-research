## 3. 任务执行流架构图

```mermaid
flowchart TD
    A[创建任务] --> B[ResearchTaskState 初始化]
    B --> C[runTask]
    C --> D[Problem Decomposer]
    D --> E[Experience Model Analysis]
    E --> F{是否开启 externalSearch}
    F -- 是 --> G[External Search]
    F -- 否 --> H
    G --> H{是否开启 Vision}
    H -- 是 --> I[Vision MoE]
    H -- 否 --> J
    I --> J{是否开启 Persona}
    J -- 是 --> K[Persona Sandbox]
    J -- 否 --> L
    K --> L[Judgment Synthesizer]
    L --> M{是否开启多模型复核}
    M -- 是 --> N[Multi-model Review]
    M -- 否 --> O
    N --> O[生成 Candidate Outputs]
    O --> P[应用 Gate]
    P --> Q[生成报告]
    Q --> R[待审核 / 已完成]
    R --> S[SQLite 持久化]
```

## 5. 一句话总结

这套架构当前可以概括为：

> **以前端工作台为入口，以任务状态机为核心，以研究编排为主干，以证据 / Vision / Persona / 体验模型为支路，以报告 / 审核 / 持久化为闭环的 AI 用研能力架构。**
