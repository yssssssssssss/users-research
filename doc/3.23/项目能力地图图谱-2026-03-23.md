# 项目能力地图图谱

日期：2026-03-23

---

## 1. 总体地图

```text
任务输入
  ├─ 文本
  ├─ 设计图 / 截图
  └─ 混合输入
       ↓
任务创建 / 持久化
       ↓
研究编排主链路
  ├─ Problem Decomposer
  ├─ Experience Model Router
  ├─ External Search
  ├─ Vision MoE
  ├─ Persona Sandbox
  └─ Judgment Synthesizer
       ↓
候选输出层
  ├─ judgment_card
  ├─ light_report
  ├─ evidence_report
  ├─ design_review_report
  └─ hypothesis_pack
       ↓
报告 / 门禁 / 审核
  ├─ Gate
  ├─ Review
  ├─ Rerun
  └─ Export
       ↓
前端工作台 / 历史任务 / 结果页 / 报告页
```

## 2. 按能力域拆分

### A. 输入能力

- 文本问题输入
- 设计图 / 截图引用输入
- 附件引用输入
- data URL / sourceUrl / ossKey 支持

### B. 编排能力

- 子问题拆解
- 体验模型自动路由
- 外部搜索支路
- 视觉评审支路
- Persona 模拟支路
- 综合判断支路
- 多模型复核支路

### C. 证据能力

- 证据池管理
- T1/T2/T3 分级
- reviewStatus 管理
- 冲突证据标记
- 待核查外部线索入池

### D. 输出能力

- 判断卡片
- 轻报告
- 证据报告
- 设计评审报告
- 假设包

### E. 治理能力

- Gate
- blockedSources
- blockedReasons
- 证据审核
- 报告审核
- rerun Vision / Persona

### F. 展示能力

- 任务中心
- 工作台
- 证据看板
- Vision Lab
- Persona Lab
- 结果页
- 报告页
- 审核页

### G. 持久化能力

- memory
- sqlite
- postgresql（设计上兼容，当前本地以 SQLite 为主）

## 5. 按任务生命周期映射

### 生命周期阶段

1. 任务创建
2. 任务运行
3. 中间状态持久化
4. 候选输出生成
5. 报告生成
6. 证据审核 / 报告审核
7. 结果留存与回放

### 每阶段对应能力

| 阶段 | 核心能力 |
|---|---|
| 创建 | 输入建模 |
| 运行 | Orchestrator |
| 持久化 | SQLite / DB |
| 输出 | CandidateOutput |
| 报告 | Report + Gate |
| 审核 | Review / Rerun |
| 回放 | 历史任务 / 当前任务恢复 |

## 7. 一句话地图总结

这个项目当前的能力图谱可以概括为：

> **以任务为中心，以编排为骨架，以证据 / 视觉 / Persona / 体验模型为支路，以输出 / 报告 / 审核 / 持久化为闭环的 AI 用研原型平台。**
