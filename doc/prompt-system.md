  关键设计原则

  1. 证据分层：所有输出严格区分 T1/T2/T3，模拟内容强制标注
  2. 模型分工：视觉评审三模型各有专项，避免重复；体验模型自动路由（复用现有 experienceModelService.ts）
  3. 可追溯：每条结论必须能回溯到具体证据来源
  4. 冲突显式化：不强行统一结论，冲突本身就是洞察
  5. 行动导向：每个模块的输出都要有具体可执行的建议，不只是描述问题
